#+TITLE: M3: Bug Fixes & Codebase Cleanup — Step Plan
#+DATE: 2026-03-15

*Branch: =m3-bugfixes= (off =m1-testing=)*

This is the first milestone that modifies =src/=. Every change runs
on production servers — work carefully, self-review, human review
on every commit. One bug at a time, verified by the test suite.

* Approach

Order: self-contained fixes first (build confidence, prove the test
suite catches things), then investigation-heavy bugs, then cosmetic
cleanup last. Each fix is one commit, tested, reviewed.

The test suite (M1) exists to verify these fixes. For each bug:
1. Confirm the test currently shows the bug (or write a test first)
2. Fix the code
3. Run tests — confirm the fix, confirm no regressions
4. Self-review the diff
5. Commit, push, human review

* Tasks

** TODO 3.1 Fix: +0M clears quota instead of no-op

*File*: =src/parse.c= line 475
*Bug*: =parse_size()= returns 0 when =count==0=, ignoring =orig=
and =op=. So =+0M= applied to 100M → 0 instead of staying at 100M.
*Test*: =test/tests/host/t-parse-units.sh= — currently shows
=KNOWN BUG - +0M clears quota=. After fix, should flip to =ok=.
*Fix*: Move the =count==0= early return after the =switch(op)=,
or handle it inside the switch (return =orig= for add/sub with 0).

/Verification/: =run-tests.sh --host-only= — known bug line disappears.
Full matrix not needed (parse_size is kernel-independent).

** TODO 3.2 Fix: "5m" timespan ambiguity passes -1 to quotactl

*Files*: =src/parse.c= =parse_timespan()= + =src/main.c= lines 139,146
*Bug*: =parse_timespan()= returns -1 on error (ambiguous "m", invalid
format), but =main.c= stores the return value directly into
=quota->block_grace= without checking for -1. The bad value reaches
=quotactl()=.
*Fix*: Check =parse_timespan()= return value in =main.c=. If -1,
print error and exit. Optionally: make bare "m" resolve to minutes
(more intuitive than error) — but that's a behavior change, discuss.
*Test*: Add a test case to =t-parse-timespan.sh= or =t-error-args.sh=
for the "5m" case once the behavior is decided.

/Verification/: =run-tests.sh --host-only=.

** TODO 3.3 Fix: -d grace field operator precedence

*File*: =src/main.c= lines 109, 123
*Bug*: =(unsigned long) quota->block_time ? ...= — cast binds only
to =block_time=, not to the ternary result. When grace has expired,
=block_time - now= is negative, but =%lu= prints it as a huge
unsigned number. BSD code path (lines 101-107) is correct.
*Fix*: Add parentheses: =(unsigned long)(quota->block_time ? ... : 0)=
Match the BSD code path's style.
*Test*: =t-dump-format.sh= checks field types. Could add an
expired-grace test (set grace very short, wait, verify -d output).

/Verification/: =run-tests.sh --host-only=.

** TODO 3.4 Remove: unused sys/cdefs.h include

*Fix*: Delete the include line. Remove the stub from

** TODO 3.5 Investigate + fix: grace -r restart (≤4.12)

*File*: =src/linux/quota.c= — =quota_reset_grace()= and/or
=xfs_reset_grace()=
*Bug*: =-r= flag sets grace to 0 instead of restarting on kernels
≤4.12. Passes on 4.15+. 7 kernels affected in the test matrix.
*Investigation needed*: What changed between 4.12 and 4.15?
Likely a kernel-side quotactl behavior change that the current
code doesn't account for. May need to probe kernel version or
try alternative quotactl commands.
*Test*: =t-grace-period.sh= already tests this. Currently fails
on ≤4.12 (known). After fix, those 7 kernels should pass.

/Verification/: Full 25-kernel matrix required. This is THE test.
The fix must pass on modern kernels (no regression) AND on ≤4.12.

*Test*: =t-grace-period.sh= skips XFS. =t-enforce-block-soft.sh=
