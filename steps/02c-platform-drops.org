#+DATE: 2026-03-19

*Status: TODO*
*Branch: =m2-1.8-development=*

* Overview

Remove code for macOS/Darwin, AIX, Solaris, and NetBSD. One commit
per platform drop (atomic, revertible). Full test suite after each.

This is surgery on the build system and conditional compilation.
Every change must leave the tree buildable and all tests passing.

Code inventory from agent research (2026-03-19) covers all files.

* Commit Strategy

Four platform-removal commits, then one build-system cleanup commit:

1. Drop macOS/Darwin
2. Drop AIX
3. Drop Solaris
4. Drop NetBSD (from =src/bsd/quota.c= + headers)
5. Build system + header cleanup (configure.ac, config.h.in, dir.mk)

Each commit: remove code → rebuild → test → commit.
Commit 5 is separate because configure.ac and config.h.in touch
all platforms at once — easier to review as one coherent change.

  interleaved =#if= chains. Triple-check preprocessor flow.
- =configure.ac= cleanup (task 5) must not break =autoconf=.
  Run =autoconf= and verify =./configure= works on Linux.
- After task 5, the intermediate host-only tests from tasks 1-4
  are superseded by the full matrix. But they catch regressions
  early while changes are small and reversible.
